# セキュリティチェックリスト

このドキュメントは、LAMP環境のセキュリティ設定を確認するためのチェックリストです。

## 📋 ネットワークセキュリティ

### VPC設定
- [ ] VPCが独立したネットワーク空間として作成されている
- [ ] VPC CIDR範囲が適切に設定されている（例：10.0.0.0/16）
- [ ] DNS解決が有効化されている（enable_dns_hostnames = true）

### サブネット設定
- [ ] パブリックサブネットとプライベートサブネットが適切に分離されている
- [ ] プライベートサブネットのリソースがインターネットから直接アクセスできない
- [ ] マルチAZ構成でサブネットが作成されている

### ルーティング設定
- [ ] パブリックサブネットのルートテーブルにInternet Gatewayが設定されている
- [ ] プライベートサブネットのルートテーブルにNAT Gatewayが設定されている
- [ ] 不要なルートが設定されていない

### セキュリティグループ設定
- [ ] **ALBセキュリティグループ**
  - [ ] HTTP (80) インバウンドが0.0.0.0/0から許可されている
  - [ ] HTTPS (443) インバウンドが0.0.0.0/0から許可されている
  - [ ] 不要なポートが開かれていない

- [ ] **Webサーバーセキュリティグループ**
  - [ ] HTTP (80) インバウンドがALBセキュリティグループからのみ許可されている
  - [ ] SSH (22) は直接許可されていない（Session Manager使用）
  - [ ] 不要なポートが開かれていない

- [ ] **データベースセキュリティグループ**
  - [ ] MySQL (3306) インバウンドがWebサーバーセキュリティグループからのみ許可されている
  - [ ] 外部からの直接アクセスが拒否されている
  - [ ] 不要なポートが開かれていない

### NACLs（Network Access Control Lists）
- [ ] デフォルトNACLsが適切に設定されている
- [ ] 追加のNACLsが必要に応じて設定されている

## 🔐 アクセス制御

### IAMロールとポリシー
- [ ] EC2インスタンス用のIAMロールが作成されている
- [ ] 最小権限の原則が適用されている
- [ ] 不要な権限が付与されていない
- [ ] Session Manager用の権限が適切に設定されている

### Session Manager設定
- [ ] AWS Systems Manager Session Managerが設定されている
- [ ] SSH接続を無効化している
- [ ] セッションログが記録されている

### キー管理
- [ ] EC2キーペアが不要な場合は作成されていない
- [ ] 必要な場合のみキーペアが使用されている
- [ ] キーペアのアクセス権限が適切に管理されている

## 🔒 暗号化

### 保存時暗号化
- [ ] EBSボリュームが暗号化されている
- [ ] データベースファイルが暗号化されている
- [ ] バックアップが暗号化されている

### 転送時暗号化
- [ ] HTTPS通信が設定されている（本番環境）
- [ ] データベース接続が暗号化されている
- [ ] 内部通信も可能な限り暗号化されている

### 証明書管理

- [ ] 証明書の有効期限が監視されている
- [ ] 証明書の自動更新が設定されている
